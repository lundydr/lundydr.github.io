Phase 0, Week 5

Classes in Ruby

January 25, 2015

Another week has gone by, and now I am officially over the Phase 0 hump - more than halfway done! Once again, this week's cirriculum focused on the Ruby language. Ruby is known as an "Object Oriented" language, or, in layman's terms, one where we perform our actions on "objects." As noted before, an object can be just about anything, and Ruby has lots of default object classes (Array, Hash, etc) built in. There's also just a default object if you are boring. But what if I want to make my own type of object that has its own set of methods I can use with it? Well, Ruby has you covered, and you can define your own classes!

Let's say we wanted to define a Class "Dog." What sorts of attributes to dogs tend to have? What would we want to include in our class? What kind of methods, or actions, would we want to do with a dog? We might want to know the dog's name, gender, age, and breed as a basic set of identifiers. In Ruby, we need to define a method "initialize" for each class that tells the computer how to handle the CLass.new call. Note that the name of the class, Dog, is capitalized, because it is a constant.

	> class Dog
	>	def initialize(name,gender,age,breed)
	>		@name = name
	>		@gender = gender
	>		@age = age
	>		@breed = breed
	>	end
	> end

Notice how all the variable declarations in the method have an @ before them? This specifies them as "instance variables," which makes them available to the entire class rather than just the normal scope of the method they are defined in. To create a new instance of the dog class we would type something like dog = Dog.new("Spaghetti","male",6,"mixed").

Okay, great, now we know some basic information about this dog, but what if we want to interact with it? We can just keep defining methods inside the class definition, and these methods will then be exclusively available to the class they are included with. Let's say we want to add two methods: feeding the dog, and petting the dog. We'll make them simple methods that just print something to the console in this case, but you could do anything you like in these methods. So after the end of the initialize block, we add:

	>	def walk
	>		puts "You take #{@name} for a nice walk around the block.  #{@name} is very happy!" 
	> 	end
	>	def feed
	>		puts "You pour some food into #{@name}\'s bowl, but he doesn\'t really care."

This code goes before the end of the class definition, in case that wasn't clear. I used a few tricks in these strings, namely inline insertion of an instance variable and escaping. When you use #{variable} inside of a string, it inserts the value of the substring variable directly into the string! Much easier than adding the strings together manually. The backslash (\) before the single quotes is an escape character, basically telling ruby I actually want to put a single quote there, not end my string.

As you can see, Classes can be very handy if you have an object type you will be using over and over in your code.
